Rejection Defense

Answers the most common fail reasons directly

No fake wallet

The live dapp uses `window.ethereum`, not localStorage simulation.

No deterministic-only contract

The contract uses `web.get`, `exec_prompt`, and validator-reviewed non-deterministic execution.

No static app-to-contract story

The repository ships deploy, write, resolve, and read-back code in both client and browser UI layers.

No silent fallback

Errors are typed, inconclusive outcomes do not auto-release, and the policy result is bound into execution.
