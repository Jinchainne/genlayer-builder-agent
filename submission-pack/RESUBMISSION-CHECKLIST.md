# Resubmission Checklist

- Contract code is present in the repository.
- The contract uses pinned GenLayer runner dependencies.
- The contract contains real non-deterministic logic that matters to the outcome.
- The validator logic checks the substance of the answer, not only its shape.
- The frontend uses a real wallet, not localStorage simulation.
- The app deploys, writes to, resolves through, and reads back from the real contract.
- The live app uses the same contract shipped in the repository.
- README explains why the use case needs GenLayer.
- Submission includes a live URL, repo URL, and ideally contract address / explorer proof.
- Reviewer notes are short, clear, and actionable.
