# GenLayer Project Fit

Use GenLayer when the product has meaningful ambiguity, judgment, evidence
evaluation, or real-world fact resolution that must affect on-chain outcomes.

Strong fit signals:

- the contract must judge evidence, language, or quality
- live web data or LLM reasoning changes stateful outcomes
- a dispute or claim needs validator-reviewed judgment
- real-world ambiguity must be persisted on-chain

Weak fit signals:

- there is no ambiguity or subjective judgment
- the app is informational only
- the chain layer is mentioned for branding, not execution

If the project does not need non-deterministic judgment, it probably does not
need GenLayer.
