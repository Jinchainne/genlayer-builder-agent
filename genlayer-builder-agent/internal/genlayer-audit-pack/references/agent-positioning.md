# Agent Positioning Reference

Use this file when deciding whether something should be described as an agent in a GenLayer project.

## Safe Uses Of The Word Agent

The project can usually be called an agent if:

- it takes action through a workflow rather than only displaying information
- it mediates evidence, policy, or judgment
- the intelligent contract is the core decision engine
- the user can submit an input and receive an adjudicated output tied to execution

Good examples:

- evidence resolution agent
- adjudication agent
- policy review agent
- dispute resolution agent

## Unsafe Uses Of The Word Agent

Avoid or narrow the wording if:

- the project is just a form
- the project is only a dashboard
- the AI component is off-chain and optional
- the contract itself does nothing meaningful

## Preferred Phrasing

Use:

- GenLayer-native adjudication agent
- intelligent-contract-based resolution agent
- evidence review agent powered by validator-reviewed consensus

Avoid:

- autonomous agent platform
- general AI agent
- agentic layer for everything
