# GenLayer Fit Reference

Use this file when deciding whether a product actually belongs on GenLayer.

## Strong Fit

A strong GenLayer project usually has all of these:

- the outcome depends on judgment
- the judgment uses evidence, language, ambiguity, or quality assessment
- the result matters to on-chain state or release of value
- multiple validators reviewing non-deterministic output is a feature, not a burden

Examples:

- evidence-backed claim resolution
- milestone completion adjudication
- grant or bounty disputes
- moderation or policy review with consequences
- reputation or responsibility assignment in ambiguous cases

## Weak Fit

A weak GenLayer project often looks like this:

- deterministic token or points logic
- a dashboard around static contract data
- AI analysis that never affects contract execution
- a UI that only simulates what a contract would do
- a project where ordinary backend infrastructure would work just as well

If the product remains a weak fit after honest review, change the concept rather than forcing GenLayer branding onto it.
